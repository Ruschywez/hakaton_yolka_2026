## Описание

Сервис представляет собой REST API с интегрированным AI-ассистентом **ВолодяGPT**  (на базе GigaChat), который помогает пользователям:

- Подбирать вакансии на основе профиля и компетенций
- Анализировать соответствие требованиям
- Строить индивидуальный карьерный путь
- Получать персонализированные рекомендации по развитию

> Какого черта вы назвали бота моим именем? (-_- )  
> Так как функционал работы с GigaChat API вынесен в отдельынй класс и
> в отдельный модуль, тенхнически это позволяет в дальнейшем заменить
> его на другой доступный сервис, например: DeepSeek.
---

## Архитектура системы

Проект состоит из двух компонентов:

1. Backend (сервер) — FastAPI-сервер с AI-ассистентом и базой данных
2. Client (клиент) — десктопное приложение на Tkinter для взаимодействия с сервером

```
┌─────────────────┐       REST API       ┌─────────────────┐
│   Клиент        │ ◄──────────────────► │   Сервер        │
│   (Tkinter GUI) │                      │   (FastAPI)     │
└─────────────────┘                      └────────┬────────┘
                                                  │
                                            ┌─────▼───────┐
                                            │   GigaChat  │
                                            │   API       │
                                            └─────────────┘
```

> Такая архитектура была не самым оптимальным выбором для хакатона.
> В условиях коротких сроков можно было выбрать и что-то проще,
> но так как, в нашей команде есть разделение по ролям было решено
> занять все руки:
> - 1 специалист по БД
> - 2 frontend разработчика
> - 1 backend разработчик
> - 1 дизайнер

## Установка

### Требования

- **Python** 3.10+
- **MariaDB** или **MySQL** сервер
- **GigaChat API** для работы AI

### Шаги установки

1. Клонируйте репозиторий:

```bash
git clone <repository-url>
cd "hakaton 2026"
```

2. Создайте и активируйте виртуальное окружение:

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux/macOS
python3 -m venv .venv
source .venv/bin/activate
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Настройте базу данных:

- Создайте базу данных `hakaton`
- Импортируйте схему (таблицы будут созданы автоматически при первом запуске)
- Отредактируйте `db_parameters.json` с вашими параметрами подключения

5. Настройте переменные окружения:

Создайте файл `.env` в корне проекта:

```env
GIGACHAT_CREDENTIALS=your_gigachat_credentials_here
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat
SYSTEM_PROMPT_FILE=system_prompt.json
```

## Настройка

### База данных

Отредактируйте `db_parameters.json`, или запустите программу и она автоматически предложит ввести данные:

```json
{
    "host": "your_host",
    "port": 3306,
    "database": "hakaton",
    "user": "your_user",
    "password": "your_password"
}
```

### GigaChat API

1. Получите API credentials в [кабинете разработчика GigaChat](https://developers.sber.ru/docs/ru/gigachat/overview)
2. Укажите credentials в файле `.env`:

```env
GIGACHAT_CREDENTIALS=ваш_ключ_авторизации
```

> **Важно:** Без настроенных credentials AI-ассистент вернет сообщение об ошибке, но базовый API **возможно** будет работать.

## API документация

### Общие сведения

- Base URL: `http://localhost:8000`
- Content-Type: `application/json`

### Авторизация и пользователи

#### **POST** `/authorization/` — Регистрация нового пользователя

**Request Body:**

```json
{
  "login": "user_login",
  "password": "secure_password",
  "last_name": "Иванов",
  "first_name": "Иван",
  "sur_name": "Иванович",
  "date_birth": "1990-01-01",
  "education_level": "Высшее",
  "education_specialice": "Информатика",
  "interests": "AI, Machine Learning"
}
```

**Response:** `200 OK` — данные созданного пользователя

---

#### **GET** `/authorization/` — Авторизация

**Query Parameters:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `login` | string | Логин пользователя |
| `password` | string | Пароль пользователя |

**Response:** `200 OK` — данные пользователя

---

#### **PATCH** `/authorization/` — Обновление профиля

**Request Body:**

```json
{
  "user_id": 1,
  "interests": "Data Science, Python"
}
```

> Обновляются только указанные поля

---

#### **DELETE** `/authorization/{user_id}` — Удаление пользователя

**Response:** `200 OK` — подтверждение удаления

---

#### **GET** `/user/{user_id}` — Получение профиля пользователя

**Response:** `200 OK` — полный профиль пользователя

---

### Сообщения и AI-ассистент

#### **POST** `/messages/` — Отправка сообщения AI-ассистенту

**Request Body:**

```json
{
  "user_id": 1,
  "text": "Какие вакансии мне подойдут?",
  "message_type": "user"
}
```

**Flow работы:**

1. Сообщение пользователя сохраняется в БД
2. Формируется контекст из профиля пользователя
3. Запрос отправляется в GigaChat
4. Ответ ассистента сохраняется и возвращается клиенту

**Response:**

```json
{
  "message_id": 42,
  "text": "На основе вашего профиля рекомендую...",
  "type": "bot",
  "date_time": "2026-04-03T12:00:00"
}
```

---

#### **GET** `/messages/` — Получение истории сообщений

**Query Parameters:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `user_id` | integer | ID пользователя |

**Response:** `200 OK` — список всех сообщений пользователя

---

#### **PATCH** `/messages/` — Редактирование сообщения

**Request Body:**

```json
{
  "message_id": 42,
  "text": "Новый текст сообщения"
}
```

---

#### **DELETE** `/messages/{message_id}` — Удаление сообщения

---

## Клиентское приложение

### Описание

Десктопное приложение с современным тёмным интерфейсом, предоставляющий удобный доступ к AI-ассистенту ВолодяGPT.

### Возможности

- Авторизация и регистрация
- Чат с AI-ассистентом в реальном времени
- Индикаторы набора текста (typing indicator)
- Профиль пользователя с просмотром и редактированием данных
- Навигация через боковую панель
- Демо-режим для тестирования без сервера

### Архитектура клиента

```
client/
├── main.py                  # Точка входа (инициализация Tkinter окна)
├── config.py                # Константы и конфигурация
├── requirements.txt         # Зависимости Python
├── api/                     # Слой взаимодействия с сервером
│   ├── __init__.py
│   ├── client.py            # HTTP-клиент (APIClient обёртка)
│   ├── auth.py              # Методы авторизации (login, register, update)
│   └── messages.py          # Методы работы с сообщениями (get, send, edit)
└── ui/                      # Пользовательский интерфейс
    ├── __init__.py
    ├── app.py               # Главный контроллер (навигация между экранами)
    ├── theme.py             # Цветовая схема и стили
    ├── components.py        # Переиспользуемые UI компоненты
    ├── login_screen.py      # Экран входа
```

### Настройка клиента

Откройте файл `config.py` и укажите адрес вашего API-сервера:

```python
# API
API_BASE_URL = "http://26.164.106.34:8000/"  # Замените на ваш адрес
MOCK_MODE = False  # True для работы без сервера (демо-режим)

# Приложение
APP_TITLE = "Ёлка — Карьерный AI Ассистент"
ASSISTANT_NAME = "ВолодяGPT"
ASSISTANT_DESCRIPTION = "Твой персональный карьерный ассистент"

# Окно
WINDOW_WIDTH = 1100
WINDOW_HEIGHT = 700
MIN_WINDOW_WIDTH = 900
MIN_WINDOW_HEIGHT = 600
```

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `API_BASE_URL` | Базовый URL сервера API | `http://26.164.106.34:8000/` |
| `MOCK_MODE` | Режим без сервера (демо-данные) | `False` |
| `APP_TITLE` | Заголовок окна | `Ёлка — Карьерный AI Ассистент` |
| `ASSISTANT_NAME` | Имя AI-ассистента | `ВолодяGPT` |
| `WINDOW_WIDTH` | Ширина окна | `1100` |
| `WINDOW_HEIGHT` | Высота окна | `700` |

```bash
python main.py
```
